What is a Struct?

A struct is a fixed-shape bundle of named fields, allocated on the stack by default, and copied every time you pass it. Once you can picture it in memory, the rest of Go gets easier.

A struct is a composite type: a recipe for a block of memory that holds a fixed set of named values, side by side. The size of that block is the sum of the sizes of its fields, plus any padding Go's allocator adds to keep fields aligned.

type User struct {
    Name   string  // 16 bytes (header + ptr + len)
    Age    int     // 8 bytes
    Active bool    // 1 byte, then 7 bytes of padding
}

That definition is a type. It produces no value by itself — it's a shape, not a thing. To get a thing, you instantiate it.

Three ways to instantiate

var u User                      // (1) zero value — all fields set to "" / 0 / false
u := User{}                     // (2) composite literal, zero value
u := User{Name: "Naveen", Age: 42} // (3) named-field literal
Always use named-field literals. Positional literals like User{"Naveen", 42, true} compile, but break silently the moment you reorder a field.

What the memory actually looks like

When you write u := User{Name: "Naveen", Age: 42} inside main, Go puts a contiguous block on the stack of the main goroutine. Stack memory is reclaimed the instant the function returns — no garbage collector involved.

Stack · main
frame for main
u User
Name → "Naveen" Age = 42 Active = true

The Name field is a string, which itself is a small struct ({ptr, len}) — the actual characters live in a read-only data section. The ptr in the struct points to that data, so a string is already a tiny pointer indirection under the hood.

Field access is just an offset

When you write u.Age, the compiler computes a constant byte offset from the start of the struct. That's why field access is the same speed whether the struct is one field or fifty.

u.Age        // read at offset 16
u.Age = 43    // write at offset 16
u.Name[0]   // "N" — follows the string's ptr

Copying is the default

This is the single fact that causes the most confusion later:

func birthday(u User) {
    u.Age++
}
func main() {
    u := User{Name: "Naveen", Age: 42}
    birthday(u)         // passes a COPY of u
    fmt.Println(u.Age)   // still 42 — original untouched
}

Stack frames show why: the function's parameter u is a separate block of memory with the same field values.

caller's frame

main
u User
Age = 42

callee's frame

birthday
u User (copy)
Age = 43

To mutate the caller's struct, the callee needs a way to reach back into the caller's memory. That way is a pointer — and that's the whole point of the next lesson.
