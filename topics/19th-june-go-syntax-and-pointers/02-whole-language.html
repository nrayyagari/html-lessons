Lesson 02 · The whole language

The whole language in one page

If you remember the 25 keywords, the four statement forms, and the way types work, you've covered the entire language surface. The rest is just a tour of the standard library.

Go's grammar is small enough to learn in an afternoon. This lesson is the whole thing — not exhaustive, but enough that you'll recognize every new thing you see as one of the patterns below.

The 25 keywords

GroupKeywords
Declarationspackage, import, const, var, type, func
Composite typesstruct, interface, map, chan
Control flowif, else, for, range, switch, case, default, break, continue, fallthrough, goto, return
Concurrencygo, select, defer
Miscnil

That's it. Notice: no while (use for), no do, no enum, no class, no extends, no implements, no new (well, it's predeclared, not a keyword), no try / catch, no public / private.

Variables and types

Three ways to declare a value. Each has a place.

// 1. var with explicit type — for zero values, package-level vars
var name string      // ""
var n   int         // 0
var ok  bool        // false

// 2. var with inferred type — when the right-hand side makes it obvious
var name = "Naveen"
var n   = 42

// 3. := — short declaration. Inside functions only.
name := "Naveen"      // preferred for local vars with an initializer
n   := 42

// 4. const — only for compile-time constants
const maxRetries = 3
const greeting = "hello"

Type inference happens in 2 and 3. The := form is the most common in real code, but you can't use it at package scope (use var there).

The four statement forms

Almost every Go statement is one of these four. The whole grammar of statements is on this page.

1. Assignment

x = 1                  // simple
x, y = y, x          // tuple (used in swaps, returns)
x++                  // post-increment only — no x + 1, no ++x
p.Field = 42          // field assignment
++ is a statement, not an expression. You can't write y := x++. This is a small thing that annoys people coming from C. The reason: it makes the grammar unambiguous in one place where it was ambiguous in C.

2. if with optional initialization

if x > 0 {
    // ...
}

// optional init statement; v is scoped to the if/else
if v := compute(); v > 0 {
    // ...
} else if v == 0 {
    // ...
} else {
    // ...
}

The init-statement form is what makes if err != nil idiomatic without a temporary leaking out:

if err := doThing(); err != nil {
    return err
}

3. for — the only loop

// 1. classic three-part
for i := 0; i < 10; i++ { ... }

// 2. while-style
for x < 100 { x *= 2 }

// 3. infinite
for { ... }

// 4. range over slice, map, string, channel
for i, v := range []int{1, 2, 3} { ... }
for k, v := range m { ... }
for i, r := range "hello" { ... }   // r is the rune, i is the byte index
for v := range ch { ... }            // v is the channel value

Note there is no while keyword. The middle form for x < 100 { x *= 2 } is the while loop. The infinite form is just for { }.

4. switch — the type-switch trick

// value switch
switch x {
case 1:
    fmt.Println("one")
case 2, 3:                  // multiple values per case
    fmt.Println("two or three")
default:
    fmt.Println("other")
}

// tagless switch — cleaner than a long if/else chain
switch {
case x < 0:
    fmt.Println("negative")
case x == 0:
    fmt.Println("zero")
default:
    fmt.Println("positive")
}

// type switch — the killer feature
switch v := i.(type) {
case int:
    fmt.Println("int:", v)
case string:
    fmt.Println("string:", v)
default:
    fmt.Println("unknown")
}
Cases don't fall through by default. Unlike C, Go's switch breaks after each matching case automatically. You don't need a break. The fallthrough keyword exists if you want C-style fall through, but it's rare and almost always a smell.

Types, in summary

Value types

bool, numeric (int/int8/.../uint64, float32/float64, complex64/complex128), string, arrays ([N]T), structs (struct{...}). All copied on assignment and function call.

Reference types

Pointers (*T), slices ([]T), maps (map[K]V), channels (chan T), functions (func(...)...), interfaces (interface{}). Hold a header that points at the underlying data. Assignment shares the backing store.

Go doesn't have a "class" system; it has types. Every named type is a first-class value. You can attach methods to any type, including types from other packages (within the same package you can't override, but you can extend).

One more thing: defer

defer is the single Go feature that has no analog in most other languages. The deferred call runs when the enclosing function returns — arguments are evaluated immediately, the call happens at return time.

func readFile(path string) error {
    f, err := os.Open(path)
    if err != nil { return err }
    defer f.Close()            // runs when readFile returns, no matter how

    // ... do stuff with f ...
    return nil
}

The pattern: acquire resource → defer release. Pairs naturally with the early-return error style. No "finally" keyword, no RAII.
