Lesson 04 · Iteration

Async iteration: async for & generators

A normal for loop calls __next__ synchronously. Some sources — a websocket, a database cursor, a stream — don't have the next value ready yet; they need to await. That's the whole point of async for.

The protocol, in one paragraph

An async iterable is anything with an __aiter__ method. __aiter__ returns an async iterator, which has an __anext__ method. __anext__ returns an awaitable that resolves to the next value, or raises StopAsyncIteration when done. async for handles all of that for you — it just calls __anext__ and waits.

This is exactly the same shape as the sync protocol (__iter__ / __next__ / StopIteration), but with await glued in.

# reading an async iterable
async for line in read_lines("huge.log"):  # awaits __anext__
    await process(line)

# building one: an async generator
async def read_lines(path):
    async with aiofiles.open(path) as f:
        async for line in f:        # aiofiles' files are async iterables
            yield line

An async def function that contains a yield is an async generator. Calling it returns an async iterator, not a coroutine. The body runs incrementally: each yield suspends the generator and hands the value to the consumer; the next __anext__ resumes after the yield.

Comprehensions have an async form too

# sync list comprehension
results = [fetch(u) for u in urls]

# async generator expression — each iteration is a coroutine, not a result
results = [fetch(u) async for u in urls_stream()]

Notice the parentheses: [... async for ...] inside [ ] builds a list of coroutines, not results. To run them and collect results, you have to await the list inside a coroutine or pass it to gather:

async def main():
    coros = [fetch(u) async for u in urls_stream()]
    results = await asyncio.gather(*coros)
The async for syntax inside comprehensions exists only in async functions. If you try [x async for x in y] outside an async def, you get a SyntaxError. The comprehension itself is async because its context is.

Async context managers: async with

The same pattern works for with: a normal with calls __enter__ / __exit__ synchronously. async with calls __aenter__ / __aexit__, which are coroutines. Use it for any resource whose acquisition or release might block on I/O — a database connection pool, an HTTP session, a lock acquired across a network.

class DbSession:
    async def __aenter__(self):
        self.conn = await connect(self.dsn)
        return self.conn

    async def __aexit__(self, *exc):
        await self.conn.close()

# use it
async def main():
    async with DbSession("postgres://…") as conn:
        rows = await conn.fetch("SELECT …")

For the common case, contextlib.asynccontextmanager turns a generator into a context manager — same idea as the sync version, but the yield happens in an async def:

from contextlib import asynccontextmanager

@asynccontextmanager
async def timed(label):
    start = time.monotonic()
    try:
        yield
    finally:
        print(f"{label}: {time.monotonic() - start:.2f}s")

async def main():
    async with timed("fetch"):
        await fetch("https://example.com")

Build a small async pipeline

Put async iteration and async context management together and you get the most useful pattern in the ecosystem — a pipeline of async generators connected by async for. Each stage yields one item at a time, so memory stays constant regardless of input size.

# producer: emit URLs from a source
async def urls_from(source):
    async for line in source:           # lines from a file, a queue, a stream
        if line.startswith("http"):
            yield line.strip()

# worker: fetch and decode
async def fetch_many(urls, concurrency=10):
    sem = asyncio.Semaphore(concurrency)
    async def one(url):
        async with sem:
            resp = await httpx.AsyncClient().get(url)
            return (url, len(resp.content))
    async for url in urls:
        yield await one(url)

# wire them up
async def main():
    async with aiofiles.open("urls.txt") as f:
        async for url, size in fetch_many(urls_from(f)):
            print(f"{url} → {size} bytes")

Each stage is a normal Python generator — testable in isolation, composable like Unix pipes. The Semaphore caps the in-flight fetches so a fast source can't flood the HTTP client.
